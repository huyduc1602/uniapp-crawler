Nâng cấp HBuilderx lên phiên bản 4.34. # Ghi chú Trong quá trình phát triển Trình cắm phát ngắn không thể bao gồm các API liên quan đến quảng cáo pangolin, bao gồm API khởi tạo của SDK quảng cáo pangolin. Các phụ thuộc liên quan đến SDK quảng cáo pangolin không thể được bao gồm trong config.json, nếu không nó sẽ gây ra lỗi bao bì. Khi đóng gói, bạn cần kiểm tra mô -đun Gromore Pangolin. SDK quảng cáo pangolin không thể được bao gồm trong trình cắm, nếu không bao bì sẽ thất bại do xung đột SDK. Để đảm bảo khởi tạo thành công các bộ phim truyền hình ngắn, bạn cần gọi bất kỳ API quảng cáo nào trước khi khởi tạo các bộ phim truyền hình ngắn Pangolin, và sau đó khởi tạo các bộ phim truyền hình ngắn Pangolin sau khi kích hoạt gọi lại AD. Ví dụ tham khảo: var kẽ xen kẽ.Onload (() => { // khởi tạo bộ phim ngắn }); xen kẽ.onerror ((err) => { // khởi tạo bộ phim ngắn }); xen kẽ.Load () Vì chương trình ngắn Pangolin hiện yêu cầu thêm địa chỉ kho lưu trữ riêng của Pangolin, HBuilderX không hỗ trợ cấu hình cho thời điểm hiện tại. Trước tiên, các nhà phát triển nên gói gọn chức năng phim truyền hình ngắn vào thư viện trong Android Studio và cung cấp một số API bên ngoài cho trình cắm UTS để gọi. Nếu bạn gặp sự cố trong quá trình phát triển trình cắm, bạn có thể đến nhóm quảng cáo UNI-AD để tham khảo ý kiến. Giúp chúng tôi cải thiện trang này!
